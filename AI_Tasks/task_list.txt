Task 2.1: Create the Codebase Indexer (Offline Script)

    File: agent/knowledge/codebase_indexer.py

    Acceptance Criteria:

        Function index_codebase(workspace_path: str, config: AgentConfig) -> Tuple[bool, str]:

            Accepts a workspace path and the agent configuration.

            Returns success status and a message (e.g., "Indexed 342 chunks successfully").

        Chunking Logic:

            Use tree-sitter to parse files with extensions: .py, .js, .ts, .java, .go, .rs (extensible list).

            For each file, split into semantic units (functions, classes, methods) using AST traversal.

            Respect .gitignore patterns (use pathspec library) to skip irrelevant files.

            Each chunk should include metadata: file_path, start_line, end_line, language.

        Embedding & Storage:

            Use sentence-transformers to load the model specified in config.rag_embedding_model.

            Compute embeddings for each chunk.

            Store in a ChromaDB collection named codebase_{workspace_hash} under rag_vector_store_path (default: .thoughtmachine/rag/codebase/).

            If a collection already exists, overwrite it or offer an --incremental flag (start with overwrite for simplicity).

        Error Handling:

            If dependencies are missing, return a clear error message.

            Log progress using the existing logging system.
